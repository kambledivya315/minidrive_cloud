Step 1: Create Appwrite Project

  1. Go to Appwrite Cloud
  2. Sign up or log in to your account
  3. Click "Create Project"
  4. Enter project name: CloudDrive
  5. Copy your Project ID (you'll need this)

Step 2: Configure Authentication

  1. In your Appwrite project, go to Auth
  2. Make sure Email/Password is enabled
  3. Add your domain to Platforms:
    • Platform Type: Web
    • Name: CloudDrive Web
    • Hostname: localhost (for development) or your domain

Step 3: Create Storage Bucket

  1. Go to Storage in your Appwrite project
  2. Click Create Bucket
  3. Bucket ID: files (or generate unique)
  4. Name: User Files
  5. Set permissions:
    • Create: users
    • Read: users
    • Update: users
    • Delete: users
  6. Copy your Bucket ID

Step 4: Update Configuration

Open js/appwrite.js and replace the following:

client.setProject('YOUR_PROJECT_ID') // Replace with your actual Project ID
const BUCKET_ID = 'YOUR_BUCKET_ID' // Replace with your actual Bucket ID

Example:

client.setProject('64f8a1b2c3d4e5f6g7h8')
const BUCKET_ID = 'files'

Step 5: Test the Application

  1. Open index.html in your browser
  2. Click "Sign Up" to create a test account
  3. Fill in the form and submit
  4. You should be redirected to the dashboard
  5. Try uploading a file to test functionality
‚úÖ Success! Your CloudDrive application should now be working with proper authentication and file management.

üîß Troubleshooting

Common Issues:

  • Login fails: Check that your Project ID is correct and the platform is configured
  • File upload fails: Verify bucket permissions and Bucket ID
  • CORS errors: Make sure your domain is added to platforms
  • Console errors: Check browser developer tools for specific error messages

For Production:

  • Use HTTPS for your domain
  • Update platform hostname to your actual domain
  • Consider implementing rate limiting
  • Set up proper backup strategies

Need help? Check the Appwrite Documentation for more details.
